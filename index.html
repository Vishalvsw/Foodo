Cafeteria Ordering System

An Interactive Project Plan

Project Introduction

This document outlines the architecture and design for a modern Cafeteria Ordering System. The application is designed to streamline the food ordering process for both customers and cafeteria staff, providing a seamless digital experience from menu browsing to order completion. The system includes features for real-time order tracking, efficient staff management, and administrative control over the menu.

Technology Stack

The system is built using a modern, scalable, and real-time technology stack to ensure a high-quality user experience and robust performance.

Database Schema

The data is structured in Firestore using collections and documents. Below is the schema for the core collections that power the application.

Collection: menu-items

Stores information for each item available on the menu.

  • name: string
  • description: string
  • price: number
  • category: string
  • imageUrl: string

Collection: orders

Contains all customer orders and their details.

  • userId: string
  • items: array
  • status: string
  • pickupSlot: string
  • timestamp: date
